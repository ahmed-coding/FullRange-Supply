## Color Palettes

### Color System Overview

Each theme includes the following color roles:

#### Primary Colors
- **Primary:** Main brand color, CTAs, key elements
- **Secondary:** Supporting brand color, accents
- **Accent:** Contrasting color for highlights

#### Semantic Colors
- **Text:** Primary text on light backgrounds
- **Text Light:** Light text on dark backgrounds
- **Background:** Default page/section background
- **Background Dark:** Alternative dark background

#### Functional Colors
- **Border:** Default border color
- **Border Light:** Subtle borders
- **Success:** Positive/success states (typically green)
- **Warning:** Warning/caution states (typically orange)
- **Error:** Error/danger states (typically red)

### Using Colors in Components

```css
/* Using CSS Variables */
color: var(--color-primary);
background: var(--color-background);
border: 1px solid var(--color-border);

/* Using Tailwind */
class="text-primary bg-accent border border-border"
```

## Spacing System

### 8px Grid

All themes use a consistent **8px spacing unit** as the foundation:

```
0px, 4px, 8px, 12px, 16px, 24px, 32px, 48px, 64px, 96px, 128px, 160px, 192px, 224px, 256px
```

Or in Tailwind multipliers:
```
0, 1 (4px), 2 (8px), 3 (12px), 4 (16px), 6 (24px), 8 (32px), 12 (48px), 16 (64px), 24 (96px), 32 (128px)
```

### Spacing Applications

| Element | Spacing |
|---------|---------|
| Padding (small) | 8px - 16px |
| Padding (medium) | 16px - 32px |
| Padding (large) | 32px - 64px |
| Margin (small) | 8px - 16px |
| Margin (medium) | 16px - 32px |
| Margin (large) | 32px - 64px |
| Gap (grid/flex) | 8px - 32px |

### Component Spacing

```jsx
{/* Small component with tight spacing */}
<div className="p-4 gap-2">...</div>  {/* 16px padding, 8px gap */}

{/* Medium component with standard spacing */}

{/* Large section with generous spacing */}
```

## Animation Guidelines

### Animation Durations

All themes follow consistent animation timing:

| Duration | Value | Usage |
|----------|-------|-------|
| Fast | 150ms | Micro-interactions, hovers |
| Base | 300ms | Standard transitions, fades |
| Slow | 500ms | Complex animations, exits |
| Very Slow | 800ms | Long-form animations, sequences |

### Easing Functions

| Theme | Easing | Character |
|-------|--------|-----------|
| Luxury Minimal | cubic-bezier(0.645, 0.045, 0.355, 1) | Smooth, refined |
| Modern Tech | cubic-bezier(0.215, 0.61, 0.355, 1) | Bouncy, energetic |
| Corporate Professional | ease-in-out | Professional, steady |
| Dark Elegant | cubic-bezier(0.645, 0.045, 0.355, 1) | Smooth, sophisticated |
| Vibrant Startup | cubic-bezier(0.215, 0.61, 0.355, 1) | Playful, bouncy |
| Industrial Edge | ease-out | Strong, direct |
| Nature Inspired | cubic-bezier(0.645, 0.045, 0.355, 1) | Natural, flowing |
| Futuristic Neo | cubic-bezier(0.68, -0.55, 0.265, 1.55) | Elastic, futuristic |
| Classic Elegant | cubic-bezier(0.645, 0.045, 0.355, 1) | Timeless, elegant |
| Bold Geometric | cubic-bezier(0.215, 0.61, 0.355, 1) | Dynamic, bold |

### Keyframe Animations

Each theme includes unique keyframe animations:

#### Luxury Minimal
- **fadeIn:** Smooth opacity transition
- **slideUp:** Content slides in from bottom
- **subtle:** Gentle floating motion

#### Modern Tech
- **slideIn:** Content slides from left
- **bounceIn:** Bouncy entrance animation
- **glow:** Glowing neon effect

#### Dark Elegant
- **fadeIn:** Smooth entrance
- **glowPulse:** Pulsing glow effect
- **neon:** Neon drop-shadow animation

#### Vibrant Startup
- **bounce:** Bouncing motion
- **spin:** Full rotation spin
- **rainbow:** Hue rotation effect

#### Futuristic Neo
- **glitch:** Glitch/corruption effect
- **scan:** Scanning lines animation
- **hologram:** Holographic effect

### Animation Usage

```jsx
{/* Entrance animation */}
<div className="animate-luxury-fade">Content fades in</div>

{/* Hover animation */}
<button className="hover:animate-tech-bounce">Click me</button>

{/* Continuous animation */}
<div className="animate-vibrant-spin">Spinning...</div>

{/* Custom animation with delay */}
<div 
  className="animate-luxury-slide"
  style={{ animationDelay: '200ms' }}
>
  Staggered entrance
</div>
```

### Animation Best Practices

1. **Keep animations short:** 150-500ms for most interactions
2. **Use easing functions:** Never use linear for organic motion
3. **Avoid excessive animations:** Only animate what matters
4. **Respect prefers-reduced-motion:** Disable for accessible use
5. **Test performance:** Animations shouldn't cause jank

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Implementation Guide

### 1. Setup Theme Provider

Wrap your app with the ThemeProvider:

```jsx
// App.jsx
import { ThemeProvider } from './context/ThemeContext'
import { LanguageProvider } from './context/LanguageContext'

export default function App() {
  return (
    <ThemeProvider>
      <LanguageProvider>
        <YourApp />
      </LanguageProvider>
    </ThemeProvider>
  )
}
```

### 2. Access Theme in Components

Use the `useTheme` hook to access theme values:

```jsx
import { useTheme, useThemeColor, useThemeSpacing } from './context/ThemeContext'

export function MyComponent() {
  const { theme, switchTheme } = useTheme()
  const primaryColor = useThemeColor('primary')
  const spacing = useThemeSpacing(2)

  return (
    <div style={{ 
      color: primaryColor,
      padding: spacing 
    }}>
      {theme.name}
    </div>
  )
}
```

### 3. Create Theme Switcher

```jsx
import { useTheme } from './context/ThemeContext'

export function ThemeSwitcher() {
  const { currentTheme, switchTheme, availableThemes } = useTheme()

  return (
    <div className="flex gap-4">
      {availableThemes.map(theme => (
        <button
          key={theme.id}
          onClick={() => switchTheme(theme.id)}
          className={`
            px-4 py-2 rounded
            ${currentTheme === theme.id 
              ? 'bg-primary text-white' 
              : 'border border-primary'
            }
          `}
        >
          {theme.name}
        </button>
      ))}
    </div>
  )
}
```

### 4. Use Tailwind with Theme

In your components, use theme-specific Tailwind classes:

```jsx
{/* Luxury Minimal Theme */}
<h1 className="text-luxury-h1 font-serif-luxury text-luxury-gold">Title</h1>

{/* Modern Tech Theme */}
<button className="bg-tech-cyan text-tech-navy rounded-tech-md">Button</button>

{/* Dynamic theme classes */}
<div className={`shadow-${theme.id}-lg rounded-${theme.id}-md`}>
  Content
</div>
```

### 5. CSS Variables Approach

Use CSS custom properties for flexibility:

```css
/* Available in all components */

border-radius: var(--radius-md);
box-shadow: var(--shadow-lg);
font-family: var(--font-serif);
animation-duration: var(--animation-duration-base);
animation-timing-function: var(--animation-easing);
```

```jsx
// Component using CSS variables
```

## Best Practices

### 1. Color Contrast

Always ensure sufficient contrast between text and background:

- **Normal text:** 4.5:1 contrast ratio
- **Large text:** 3:1 contrast ratio
- **UI components:** 3:1 contrast ratio

Use the [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) to verify.

### 2. Typography Hierarchy

- Use headings semantically (H1, H2, H3, etc.)
- Maintain consistent line length (45-75 characters for body text)
- Use font weights to create hierarchy, not just size
- Ensure sufficient line height for readability (1.5-1.8)

### 3. Spacing Consistency

- Use the 8px grid consistently
- Maintain consistent gaps within components
- Use padding for internal spacing, margin for external
- Group related elements with tighter spacing

### 4. Animation Best Practices

- Keep animations under 500ms for UI interactions
- Use easing functions appropriate to the theme
- Respect `prefers-reduced-motion` for accessibility
- Test on actual devices, not just fast computers
- Avoid animating properties that trigger layout recalculation

### 5. Accessibility

- Ensure all interactive elements are keyboard accessible
- Use proper ARIA labels for screen readers
- Provide alt text for all images
- Use semantic HTML (buttons not divs for actions)
- Test with accessibility tools (axe, Lighthouse)

### 6. Performance

- Minimize animation CPU usage (use `transform` and `opacity`)
- Defer loading of non-critical resources
- Optimize images and use appropriate formats
- Use CSS Grid/Flexbox for responsive layouts
- Test Core Web Vitals (LCP, INP, CLS)

### 7. Responsive Design

- Mobile-first approach
- Use Tailwind's responsive prefixes (sm:, md:, lg:, xl:)
- Test on multiple device sizes
- Maintain touch targets (min 44x44px)
- Adapt spacing for mobile devices

### 8. Theme Switching

- Save user preference to localStorage
- Maintain consistent layout during theme switch
- Preload theme colors to avoid flash
- Provide clear visual feedback of current theme
- Allow theme preference based on system settings

### 9. Color Usage

- Don't rely on color alone to convey information
- Use semantic colors consistently
- Maintain sufficient contrast in all states
- Test with color-blind vision simulators
- Consider dark mode impact on all colors

### 10. Component Reusability

- Create components for each design pattern
- Use props to customize theme-specific values
- Document props and usage examples
- Maintain consistent component API
- Version components for backward compatibility
